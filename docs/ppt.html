🔥 热搜 Top 10 — 为什么不用 SQL 排序？

❌ SQL 方案

SELECT * FROM articles
ORDER BY score DESC
LIMIT 10
  • ⏱ O(N log N) 全表排序
  • 💀 10 万文章 ≈ 200ms
  • 🔄 每次查询扫全表

✅ ZSET 方案

ZREVRANGE hot_articles
0 9 WITHSCORES
  • ⚡ O(log N) 有序集合
  • 🚀 10 万文章 ≈ 1ms
  • 💪 单机 10 万 QPS

消息驱动架构 — RabbitMQ 拓扑详解

📨 article.topic.exchange
TopicExchange · durable=true · autoDelete=false
routing_key: article.publish
article.tag.queue binding: article.publish
↓
TagExtractConsumer
5 并发线程
article.compliance.queue binding: article.publish
↓
ComplianceCheckConsumer
10 并发线程

同一条 routing_key 绑定两个 queue → Exchange 同时投递 → fan-out 效果

@Bean public TopicExchange articleTopicExchange() { return new TopicExchange( "article.topic.exchange", true, // durable false // autoDelete ); } // Two queues bound with article.publish // routing key → fan-out effect

🐰 RabbitMQ

  • ✅ Topic 灵活路由
  • ✅ 内置管理 UI
  • ✅ 低运维成本
  • ✅ Spring AMQP 深度集成

📨 Kafka

  • 高吞吐（百万级/秒）
  • 日志收集 & 流处理
  • ZooKeeper / KRaft 重运维
  • 分区有序性保证
💡
选型结论：当前场景 QPS < 1000 → RabbitMQ 最佳选择；QPS > 10K 时迁移 Kafka
